# DB FleetOps 프로젝트 헌장

## 1. 프로젝트 이름

DB FleetOps

Multi-Database Operations Control Plane

## 2. 프로젝트 목적

여러 서버에 분산된 데이터베이스 인스턴스를 등록하고 상태를 진단하며,
백업과 설정 변경 같은 운영작업을 안전한 비동기 Job으로 실행하고,
모든 요청과 변경 이력을 추적할 수 있는 DBMS 운영관리 플랫폼을 개발한다.

## 3. 해결하려는 문제

- DB 인스턴스 정보가 여러 문서와 담당자에게 분산되어 있다.
- 운영자가 서버에 직접 접속하여 SQL과 명령어를 실행한다.
- 백업과 설정 변경 작업의 진행 상태와 결과가 체계적으로 남지 않는다.
- 동일 작업이 중복 실행될 수 있다.
- DBMS마다 조회 방법과 운영 명령이 다르다.
- 장애 대응이 담당자의 경험에 의존한다.

## 4. 핵심 목표

- 여러 DB 인스턴스를 하나의 API로 관리한다.
- DBMS별 차이를 Adapter로 분리한다.
- 조회 작업과 변경 작업을 구분한다.
- 위험한 변경 작업은 비동기 Job으로 실행한다.
- 작업 중복과 Worker 장애에 대응한다.
- 모든 작업 요청과 결과를 Audit Log로 기록한다.
